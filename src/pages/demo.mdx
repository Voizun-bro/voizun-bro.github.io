    ## TL;DR

    - Use VSCode workspace settings (configured per project)
    - Recommended for teams using VSCode
    - This example focuses on Prettier configuration

    ## Configuration Steps

    ### 1. Extension Settings

    Configure extensions in `.vscode/extensions.json`:

    <Code code={`{
      "recommendations": [
        "esbenp.prettier-vscode"
      ]
    }`} lang="json" theme="dark" />

    ### 2. Workspace Settings

    Configure VSCode settings in `.vscode/settings.json`:

    <Code code={`{
      "editor.formatOnSave": true,
      "editor.defaultFormatter": "esbenp.prettier-vscode",
      "[javascript]": {
        "editor.defaultFormatter": "esbenp.prettier-vscode"
      },
      "[typescript]": {
        "editor.defaultFormatter": "esbenp.prettier-vscode"
      }
    }`} lang="json" theme="dark" />

    ## Target Audience

    - VSCode v1.52.1 or later
    - Teams using VSCode for collaborative development

    ## Benefits

    1. **Consistency**: Ensures all team members use the same formatting rules
    2. **Automation**: Automatic formatting on save
    3. **Version Control**: Settings can be committed to the repository
